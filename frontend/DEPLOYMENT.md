# Развертывание статического лендинга

## Характеристики

Это одностраничный статический лендинг (SPA), который:
- ✅ Рендерит контент СРАЗУ при загрузке
- ✅ Не использует react-router
- ✅ Не имеет условных рендеров блокирующих контент
- ✅ Не использует backend/API
- ✅ Работает в поддиректории сайта (относительные пути)
- ✅ body и #root имеют min-height: 100vh

## Сборка для продакшена

```bash
cd /app/frontend
yarn build
```

Результат сборки будет в папке `build/`:
- `index.html` - главная страница
- `static/` - CSS и JS файлы

## Проверка сборки

1. Откройте `build/index.html` в браузере
2. Убедитесь, что `<div id="root">` содержит контент
3. Проверьте, что все изображения загружаются

## Развертывание

### Вариант 1: Загрузка на хостинг
Скопируйте содержимое папки `build/` на ваш хостинг.

### Вариант 2: Поддиректория
Лендинг уже настроен для работы в поддиректории благодаря `"homepage": "."` в package.json.

Пример: если разместить в `/landing/`, файлы будут доступны:
- `/landing/index.html`
- `/landing/static/css/...`
- `/landing/static/js/...`

### Вариант 3: Локальная проверка
```bash
cd build
python3 -m http.server 8000
```
Откройте http://localhost:8000

## Структура файлов

```
build/
├── index.html              # Главная страница
├── asset-manifest.json     # Манифест ресурсов
└── static/
    ├── css/
    │   └── main.*.css      # Стили
    └── js/
        └── main.*.js       # JavaScript
```

## Важно

- Все пути относительные (`./static/...`)
- Контент отображается сразу, без условий инициализации
- Нет зависимостей от backend
- useEffect используется только для анимаций (не блокирует рендеринг)
